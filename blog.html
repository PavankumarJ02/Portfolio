How I started this portfolio project and the process behind building it.

This portfolio began as a simple goal: create one clear place to present my work, technical interests, and growth as a Computer Science student. I wanted the site to feel professional, direct, and easy for recruiters, mentors, and collaborators to understand in just a few minutes.

01

Start

I started this project because I needed more than a resume file. A resume gives short points, but it does not fully show how I think, what I build, or the kind of engineer I want to become. This site was my way of turning scattered information into a focused story about my work.

02

Thinking

The first decision was not code. It was clarity.

What I wanted the site to communicate

  • I am serious about AI, machine learning, and practical software systems.
  • I can explain projects clearly instead of listing tools without context.
  • I care about clean presentation, not only technical implementation.
  • I am still growing, but I already build with intent and structure.

Core idea

Instead of making the page crowded with every detail, I focused on showing the strongest parts first: selected work, capabilities, learning journey, and contact points.

03

Process

How I moved from idea to a finished portfolio.

Step 01

Collected the content

I gathered my project details, education, certifications, technical skills, and contact links. This helped me understand what should be included and what was better left out.

Step 02

Planned the structure

I organized the page into clear sections so a visitor could scan it quickly: hero, profile, work, capabilities, journey, certifications, and contact.

Step 03

Chose a visual direction

I used a minimal three-color system with strong type contrast. The intention was to make the site look calm and professional while still having enough identity to stand out.

Step 04

Built and refined the experience

I added a responsive layout, subtle motion, a dark mode toggle, and cleaner spacing. Then I kept refining the content so each section said something useful instead of filling space.

Project 01

Jul 2024 - Dec 2024

Real-Time Health and GPS Tracker for Climbers

This project started from a practical safety problem. Climbers in remote locations can face sudden health risks, route changes, or communication issues, and I wanted to build a system that could monitor critical signals and send alerts early.

  • I combined health tracking and GPS monitoring into one safety-focused workflow.
  • I used Raspberry Pi and Arduino UNO to handle sensing, monitoring, and device-level coordination.
  • I worked on low-power thinking because remote use is not useful if the device drains quickly.
  • I explored SVM-based analysis to better separate and study temperature and blood pressure patterns.

What mattered most in this project was not only collecting data, but making the system meaningful in a real environment. That pushed me to think about reliability, alerts, and how technical choices connect to user safety.

Project 02

May 2025 - Jul 2025

Music Genre Classification and Language Detection System

I built this project because it let me work at the intersection of machine learning, audio analysis, and language understanding. I wanted to create something that did more than classify a song, and instead combined genre recognition with lyric language detection in one pipeline.

  • I used Mel spectrogram features with Librosa to convert raw audio into useful model input.
  • I built a CNN with TensorFlow and Keras for multi-class genre classification.
  • I added OpenAI Whisper to transcribe audio and identify lyric language from the result.
  • I used accuracy and loss tracking to evaluate model behavior and guide improvements.

This project helped me better understand how feature extraction, model selection, and evaluation connect across an end-to-end ML workflow. It also pushed me to think about how different AI tools can be combined into one useful system.
